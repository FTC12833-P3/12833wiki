camera streaming


To stream your camera to the dashboard you will first need to set up a vision portal. This is the vision portal builder

visionPortal = new VisionPortal.Builder()

.setCamera(opMode.hardwareMap.get(WebcamName.class, "Webcam 1")) // < Here you specify your webcam which you get from the hardwareMap

.addProcessor(aprilTagProcessor) // < This is a addProccessor function, same asthe line immediately below. These add different proccessors that you may or may not need

.addProcessor(cameraStreamProcessor) // < If you just use the camera for streaming you will only need this processor

.setStreamFormat(VisionPortal.StreamFormat.MJPEG) // < setting the stream format mpeg is the reccomended format as it is lightweight

.setAutoStopLiveView(true)// < This is in case the robot stops, you won't get extra errors

.build(); // < Build is pretty self explanitory it builds the new vision portal with the specifications you entered above

Then you will need to add this line:

FtcDashboard.getInstance().startCameraStream(cameraStreamProcessor, 0);

^This line gets the instance then starts the camera streaming using the cameraStreamProcessor that we created above. The zero is the max FPS 0 means there is no limit

The camera should stream when you have dashboard open and the robot running

NOTE: This will not show the apriltag / tensorflow / openCv bounding boxes you will need to use scrcpy (screen copy)
